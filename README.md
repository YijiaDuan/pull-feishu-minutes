# pull-feishu-minutes.skill

> *"会开完了，讲座听完了，Coffee Chat 聊爽了。录音躺在飞书云端，本地知识库里一个字都没有。"*

**不用自建飞书应用，不用申请 API 权限，不用求管理员审批——在弹出的浏览器里登录一次，剩下的全自动。**

跟 Claude 说一句「拉一下我的飞书妙记」，它会拉起一个浏览器让你登录飞书，然后把「我的妙记」里的**全部**录音转写抓到本地，每条存成一篇 Markdown：**顶部是它替你写的总结，底部是完整的原始逐字稿**。首次全量，之后每次只拉新增的。

诚实的代价：飞书开放 API 给不了这件事（原因见下），所以本 skill 走的是妙记网页版的内部接口。好处是零配置门槛，坏处是**飞书改版有失效可能**。失效时脚本会明确报错，不会假装成功——这一点在设计上是刻意保证的。

## 安装

> **提示**：Claude Code 会从 `~/.claude/skills/`（全局）或当前 git 仓库根目录的 `.claude/skills/`（项目级）自动加载 skill。

```bash
# 全局（所有项目都能用）—— 推荐
git clone https://github.com/YijiaDuan/pull-feishu-minutes \
  ~/.claude/skills/pull-feishu-minutes

# 或只装到当前项目
mkdir -p .claude/skills
git clone https://github.com/YijiaDuan/pull-feishu-minutes \
  .claude/skills/pull-feishu-minutes
```

首次使用时 Claude 会自动跑 `scripts/setup.sh`，建好 venv、装 Playwright、下 Chromium 内核。你不用手动装任何东西。

**依赖**：Python 3.10+、能访问飞书的网络。就这些。

## 使用

装好之后，直接跟 Claude Code 说人话就行：

```
拉一下我的飞书妙记
```

以下说法都能触发：

- 「把我的妙记同步到本地」
- 「同步飞书妙记到我的 Obsidian 库」
- 「导出飞书妙记」
- 「我这周开的会录音都拉下来」
- 「sync my feishu minutes」

**第一次运行**会弹出一个浏览器窗口（标题是 *Google Chrome for Testing*，不是你平时的 Chrome）。点右上角「注册/登录」，用手机飞书扫码即可。登录成功后脚本自动继续，你不需要复制 cookie、不需要碰开发者工具、不需要任何技术操作。

**之后再运行**不会再弹窗——登录态存在 `~/.config/feishu-minutes/browser-profile`，静默跑完。

## 它做什么

```
   你说「拉一下飞书妙记」
            │
            ▼
   ┌────────────────────┐
   │ 弹浏览器 → 你扫码   │  ← 只有第一次
   └────────┬───────────┘
            ▼
   列出「我的妙记」全部条目（分页拉完，不漏）
            │
            ▼
   跟本地已有的比对 ──► 已拉过的跳过
            │
            ▼
   逐条导出逐字稿（带说话人 + 时间戳）
            │
            ▼
   Claude 通读每篇 ──► 写总结、判断场景、重命名
            │
            ▼
   落地成 Markdown
```

每篇笔记长这样：

| 部分 | 内容 |
|---|---|
| **frontmatter** | 重写后的标题、`场景`（Coffee Chat / 讲座 / 会议…）、tags、日期、时长、说话人数、飞书原链接、`minute_token` |
| **这是哪一场** | 为「一个月后已经忘光的自己」写的场景还原：什么活动、谁在讲、现场发生了什么、一句话记忆点 |
| **沉淀下来的认知** | 这场真正有价值的东西，带具体数字、案例、原话里的犀利表述 |
| **可借鉴的 idea** | 能直接拿来用的几条 |
| **原始逐字稿** | 完整保留，一字未删 |

Claude 写总结时会**主动过滤掉**寒暄、设备故障吐槽、他人八卦，以及涉及个人隐私的内容（离职、健康、薪资、感情等）——Coffee Chat 里这类内容往往不少，你大概率不想让它们进知识库。

### 可选：补转飞书没转的妙记

飞书免费版只有 **300 分钟** 转写额度，超额的录音会没有逐字稿、或只转了开头一小段。配好一个语音大模型的 key 后，这类妙记会被自动识别出来，把音频抠出来重新转写（带说话人分离和时间戳），补进同样的 Markdown。

- **默认关闭**，不配 key 完全不影响主功能——没转写的妙记会被跳过并在结果里列出来。
- 目前接的是**阿里云百炼 Paraformer**：转 1 小时音频约几毛钱、1~2 分钟出结果。
- 需要的环境变量：`DASHSCOPE_API_KEY` + 一组 OSS 凭证（音频要经对象存储中转，百炼才够得到；音频进的是你自己的私有 bucket，用完即删）。详见 [SKILL.md](SKILL.md) 的「可选：ASR 兜底」。
- 判断"需要补转"看的是**转写覆盖率**而不是"有没有字"——额度耗尽时飞书常常已经转了开头两三句，只看有没有字会漏判。

## 为什么不用开放 API

这不是偷懒，是走过弯路之后的结论：

| | 飞书开放 API | 本 skill |
|---|---|---|
| **能列出我全部妙记吗** | ❌ 没有列表接口。只有一个关键词搜索接口，按相关度返回少量命中，**枚举不全** | ✅ 分页拉全，一条不漏 |
| **导出逐字稿要什么** | 敏感权限 `minutes:minutes.transcript:export` | 登录一次 |
| **需要自建应用吗** | ✅ 要，还要配 App ID / Secret | ❌ 不需要 |
| **需要管理员审批吗** | ✅ **要企业管理员在管理后台单独放行** | ❌ 不需要 |
| **普通员工能自己搞定吗** | ❌ 你多半不是管理员 | ✅ 能 |
| **稳定性** | 官方接口，长期稳定 | 内部接口，改版有失效风险 |

一句话：开放 API 那条路，**光是"列出我有哪些妙记"这一步就走不通**，再加上导出权限卡在管理员审批，对绝大多数人是死路。

## 适合 / 不适合

| ✅ 适合 | ❌ 不适合 |
|---|---|
| 想把自己的会议/讲座/Coffee Chat 录音沉淀进本地知识库 | 想批量下载**别人**分享给你的妙记（只拉「我的妙记」） |
| Obsidian / Logseq 等本地 Markdown 笔记用户 | 想要音视频原文件（本 skill 只拉文字） |
| 受不了每次手动复制粘贴逐字稿 | 需要 7×24 无人值守的生产级同步（登录态会过期） |
| 希望录音自动带上 AI 总结，而不是一堆生文本 | 你的妙记是别人企业空间里的、你没有查看权限 |
| 没有飞书管理员权限的普通员工 | 需要官方 SLA 保障的商业集成 |

## 坑（都是实测踩出来的）

- **弹出的浏览器和你平时的 Chrome 是隔离的两套配置**。你在日常 Chrome 里已登录飞书，对这里不起作用——这是设计如此，也正是「没授权过飞书的人也能用」的前提。
- **首次运行必须有界面**。加 `--headless` 就没窗口，你无从登录，只会干等到超时。
- **未登录时飞书的列表接口也返回"成功、空列表"**，跟「已登录但没有妙记」完全无法区分。所以本 skill 靠页面上有没有登录入口来判断，并且拉到 0 条时会**明确警告**，绝不静默假成功。
- **等待登录时脚本不会刷新页面**。刷新会把你正在扫的码冲掉——这个坑踩过，别再踩。
- **登录后飞书会把你重定向到企业专属域名**（`xxx.feishu.cn`），不再是 `meetings.feishu.cn`。所有请求必须跟着当前域走，跨域会被 CORS 直接拦死。
- **导出接口每个域名各有一份不同的 CSRF token**，取错域一律 HTTP 400。
- **改文件名、移动文件都是安全的**。去重认的是 frontmatter 里的 `minute_token`，不是文件名。
- **登录态会过期**。过期后重跑一次、重新扫码即可，已拉过的不会重复拉。

## 项目结构

```
pull-feishu-minutes/
├── SKILL.md              # 给 AI 看的执行手册（Claude Code 读这个）
├── README.md             # 给人看的（你正在读）
├── README_EN.md
├── LICENSE
└── scripts/
    ├── setup.sh          # 建 venv + 装 Playwright + 下 Chromium
    └── sync_minutes.py   # 登录 / 枚举 / 导出 / 增量去重
```

**刻意没有的东西**：没有 `prompts/`（写总结的指令直接写在 SKILL.md 里，因为它就是给 AI 的指令）；没有 `requirements.txt`（唯一依赖是 Playwright，`setup.sh` 一行装完）；没有把 `.venv` 提交进仓库（147MB，本地生成即可）。宁可结构朴素，不做撑场面的空目录。

`SKILL.md` 里还有一节 **「给维护者：绝对不要改坏的几处」**，把每一个会导致「静默假成功」的坑都记下来了。要改 `sync_minutes.py` 之前建议先读。

## FAQ

**Q：Windows / Linux 能用吗？**
理论上能——Playwright 三平台通吃，代码里没有 macOS 专属逻辑。但作者只在 macOS 上实测过，欢迎提 issue 反馈。

**Q：会拉别人分享给我的妙记吗？**
不会。只拉「我的妙记」（你自己的空间）。共享空间的内容不在范围内。

**Q：能拉音频/视频原文件吗？**
不能，只拉文字转写。想要媒体文件的话，妙记页面上有下载入口。

**Q：能做成定时自动同步吗？**
脚本本身支持 `--headless` 静默运行，挂 cron / LaunchAgent 技术上可行。但要注意两点：登录态会过期（届时任务会失败并留下日志）；写总结这一步需要 Claude 在场，无人值守时只会落下裸逐字稿。

**Q：安全吗？会不会把我的登录信息传给第三方？**
不会。登录发生在你本机的浏览器里，凭证只存在本机 `~/.config/feishu-minutes/browser-profile`，脚本只跟 `feishu.cn` 通信，不上传任何东西到别处。代码全部开源，可以自己审。

**Q：飞书改版失效了怎么办？**
脚本会明确报错而不是假装成功。SKILL.md 里记录了所有内部接口的细节和坑，照着修不难。也欢迎提 issue 或 PR。
